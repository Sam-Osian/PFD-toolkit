Privacy Policy

This notice explains how Orelian Ltd handles personal data when you use PFD Toolkit to browse the website, sign in, create workbooks, run investigations, or use email notifications.

Who We Are

Orelian Ltd operates PFD Toolkit and is the controller for personal data processed through this service. PFD Toolkit is the product name, not a separate legal entity. For privacy questions, email sam.osian@oreliandata.co.uk.

Information We Process

Category Examples
Account data Email address, first name, last name, Auth0 account identifier, account status and admin flags.
Authentication and session data Session identifiers, CSRF tokens, OAuth state values, and saved Explore filter state in the server session.
Workbook and investigation data Workbook titles, descriptions, membership records, user roles, investigation titles/questions, method settings, report exclusions, revision history, share links, and public/private visibility choices.
Run and artifact data Run configuration, status, progress events, errors, generated CSV/ZIP artifacts, artifact metadata, and timestamps.
LLM configuration and credentials Selected provider, model name, concurrency setting, optional base URL, encrypted API keys, last four API key characters, and credential use timestamps.
Audit, security, and operational data Action logs, hashed IP addresses, user agent strings, rate-limit identifiers, worker health records, slow request/query logs, and email notification records.
Public-source PFD report data Published Prevention of Future Death report metadata and text used for search, filtering, thematic analysis, and generated outputs. Information that relates only to a deceased person is outside UK GDPR, but report content may still include information about identifiable living people such as officials, respondents, or report recipients.
Browser storage data Local draft wizard content, tutorial/notice acknowledgement flags, and strictly necessary cookies described in the Cookie Policy.

Why We Use It

  • To provide the website, account login, workbooks, investigations, sharing, downloads, and notification features.
  • To authenticate users, manage access rights, secure the service, prevent misuse, and maintain audit trails.
  • To run user-requested workflows over public PFD report data and produce artifacts for the requesting user or workspace members.
  • To store user-selected LLM provider settings and encrypted credentials so workflows can run without repeatedly pasting API keys.
  • To operate, debug, maintain, and improve reliability and performance.
  • To comply with legal obligations where applicable.

Lawful Bases

Depending on the processing purpose, we rely on one or more UK GDPR lawful bases: performance of a contract or steps requested by you to provide the service; legitimate interests in operating, securing, improving, and supporting the service; consent where we ask for it; and legal obligation where the law requires processing.

Who We Share Data With

We use service providers for authentication, hosting, database/storage infrastructure, email delivery, and operational support. Authentication is handled through Auth0. Email notifications may be sent through the configured SMTP provider. Run artifacts may be stored in S3-compatible object storage in production.

When you run an LLM workflow, relevant prompt text, selected PFD report content, and workflow instructions are sent to the LLM provider you select, currently OpenAI or OpenRouter. Your API key is used to authenticate to that provider and is not intentionally displayed after saving.

International Transfers

Some providers may process data outside the UK. Where this happens, transfers need appropriate safeguards such as provider terms, data processing agreements, adequacy arrangements, or transfer mechanisms. Orelian Ltd is reviewing these arrangements for the providers used to operate PFD Toolkit.

Retention

Data Current retention approach in the app
Accounts and membership data Kept while the account or workspace membership is active, unless deletion is requested and no overriding reason requires retention.
Workbooks, investigations, revisions, and share links Kept until deleted by an authorised user or lifecycle maintenance applies. The configured inactivity window is 365 days; inactive workbooks may be archived, then purged after a further 60 days.
Run artifacts Created with a 365-day retention/expiry window. Human views refresh a sliding inactivity date; lifecycle maintenance marks stale artifacts as expired. Deletion of the underlying file or object-storage item depends on the configured storage backend and operational lifecycle rules.
LLM API credentials Stored encrypted until replaced, deleted by the user, or deleted with the related account/workspace data.
Audit and security records Kept as needed for security, integrity, support, and accountability. Some audit records may remain after a workbook is deleted.
Browser storage Stored on your device until the workflow clears it or you clear browser storage. Wizard drafts are cleared when a launch succeeds.

Your Rights

If UK data protection law applies, you may have rights to access, rectify, erase, restrict, object to processing, and receive certain data in a portable format. Some rights depend on the lawful basis and are not absolute.

You can raise a privacy request using sam.osian@oreliandata.co.uk. You also have the right to complain to the UK Information Commissioner's Office.

Automated Decisions

Orelian Ltd makes LLM-assisted analysis available in PFD Toolkit to help users analyse public PFD report content. The service does not use automated processing to make legal or similarly significant decisions about users.

Changes

We may update this policy when the service, providers, retention settings, or legal requirements change. The current version will be published on this page.
